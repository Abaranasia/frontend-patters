Observer Pattern

This patter give us a way to control the behavior of an object depending of an event

The Observer Pattern is based on a Subject class with an attribute (notify) and two methods (subscribe and unsubscribe) that will attend the events (or stop doing it) that are triggered from other objects. These objects can shoot different states or functions in the subject, which will be notified when shooted.

The aim of this pattern is to be able to update the state of the subject in as many ways as needed from different events while it is subscribed
